# Добрый Городок 🏙️

Привет! Это репозиторий проекта «Добрый Городок». Проект представляет собой монорепозиторий, включающий клиентскую часть (фронтенд) и серверную часть (бэкенд). 
В качестве базы данных используется облачное решение **Supabase**, поэтому поднимать локальную БД не требуется.

---

## 🛠 Требования к окружению

Перед началом работы убедитесь, что на вашем компьютере установлены:
* **[Node.js](https://nodejs.org/)** (рекомендуется LTS версия 18.x или выше)
* **[Git](https://git-scm.com/)**

---

## 🚀 Как запустить проект локально

### 1. Клонирование репозитория
Откройте терминал, выберите нужную папку и выполните команды:

git clone https://github.com/ВАШ_ЛОГИН/dobriy-gorodok.git
cd dobriy-gorodok

*(Не забудьте заменить ссылку на актуальную)*

### 2. Установка зависимостей
Благодаря настройке монорепозитория, вам не нужно устанавливать пакеты по отдельности. Выполните в корневой папке одну команду:

npm run install:all

Она автоматически подтянет все нужные node_modules и для фронтенда, и для бэкенда.

### 3. Настройка переменных окружения (.env)
Проекту нужны ключи от базы данных и API, которые мы не храним в открытом доступе на GitHub. Вам нужно создать два файла .env.
> **Важно:** Запросите актуальные значения ключей у владельца репозитория.

**В папке backend/ создайте файл .env:**
PORT=5000
DATABASE_URL=ваш_url_подключения_к_postgres
JWT_SECRET=ваш_секретный_jwt_ключ
TG_TOKEN=ваш_токен_телеграм_бота
TG_CHAT_ID=ваш_id_чата

**В папке frontend/ создайте файл .env:**
VITE_API_URL=http://localhost:5000
VITE_SUPABASE_URL=ваш_url_supabase
VITE_SUPABASE_KEY=ваш_публичный_anon_ключ_supabase

*(Если фронтенд работает на Create React App, замените префиксы VITE_ на REACT_APP_)*

### 4. Запуск проекта
Убедитесь, что находитесь в **корневой папке** (dobriy-gorodok), и запустите проект:

npm run dev

Оба сервера запустятся одновременно:
* **Бэкенд** будет доступен на http://localhost:5000
* **Фронтенд** откроется по ссылке, которую выдаст терминал (обычно это http://localhost:5173 или http://localhost:3000)

## 📝 Разработка и структура

* frontend/ — клиентская часть приложения (React).
* backend/ — серверная часть приложения (Node.js/Express).
* При добавлении новых пакетов обязательно переходите в соответствующую директорию (cd frontend/ или cd backend/), а не устанавливайте их в корне.
* В проекте работает «горячая перезагрузка» (hot-reload): изменения в коде сразу применяются без ручного перезапуска сервера.
